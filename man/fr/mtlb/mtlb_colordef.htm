mtlb_colordef - Fonction d'émulation de la fonction Matlab colordef

Description

Il n'y a pas de fonction Scilab équivalente à la fonction Matlab colordef mais il existe des instructions équivalentes.

La fonction h = mtlb_colordef(color_option) ou h = mtlb_colordef(fig,color_option) est utilisée par mfile2sci pour remplacer colordef(color_option) ou colordef(fig,color_option) quand il n'était pas possible de savoir ce qu'étaient les paramètres d'entrée pendant la conversion d'un code Matlab vers Scilab. Cette fonction va déterminer la sémantique correcte pendant l'exécution. Pour obtenir un code plus performant on peut remplacer les appels à mtlb_colordef :

  • Pour les appels avec un paramètre d'entrée, si color_option est égal à "black" ou "none" mtlb_colordef(color_option) peut être remplacé par fig = gcf();fig.background = -1;
  • Pour les appels avec un paramètre d'entrée, si color_option est égal à "white" mtlb_colordef(color_option) peut être remplacé par fig = gcf();fig.background = -2;
  • Pour les appels avec deux paramètres d'entrée, si fig est un handle graphique et color_option est égal à "black" ou "none" mtlb_colordef(color_option) peut être remplacé par fig.background = -1;
  • Pour les appels avec deux paramètres d'entrée, si fig est un handle graphique et color_option est égal à "white" mtlb_colordef(color_option) peut être remplacé par fig.background = -2;
  • Pour les appels avec deux paramètres d'entrée, si fig est égal à "new" et color_option est égal à "black" ou "none" mtlb_colordef(color_option) peut être remplacé par fig = scf(max(winsid())+1);fig.background = -1;
  • Pour les appels avec deux paramètres d'entrée, si fig est égal à "new" et color_option est égal à "white" mtlb_colordef(color_option) peut être remplacé par fig = scf(max(winsid())+1);fig.background = -2;
  • Pour les appels avec un paramètre de sortie h, ajouter h = fig; à la suite des instructions équivalentes.

    • Attention : mtlb_colordef ne doit pas être utilisée pour une programmation manuelle.
