mtlb_repmat - Fonction d'émulation de la fonction Matlab repmat

Description

Il n'y a pas de fonction Scilab équivalente à la fonction Matlab repmat mais il existe des instructions équivalents.

La fonction mtlb_repmat(M,m[,n]) est utilisée par mfile2sci pour remplacer repmat(M,m[,n]) quand il n'était pas possible de savoir ce qu'étaient les paramètres d'entrée pendant la conversion d'un code Matlab vers Scilab. Cette fonction va déterminer la sémantique correcte pendant l'exécution. Pour obtenir un code plus performant on peut remplacer les appels à mtlb_repmat :

  • Si A est de type Double et que m est un scalaire, mtlb_repmat(M,m) peut être remplacé par ones(m,m).*.M et mtlb_repmat(M,m,n) peut être remplacé par ones(m,n).*.M
  • Si A est de type Booléen et que m est un scalaire, mtlb_repmat(M,m) peut être remplacé par ones(m,m).*..bool2s(M) et mtlb_repmat(M,m,n) peut être remplacé par ones(m,n).*..bool2s(M)
  • Si A est de type String et que m est un scalaire, mtlb_repmat(M,m) peut être remplacé par asciimat(ones(m,m).*.asciimat(M)) et mtlb_repmat(M,m,n) peut être remplacé par asciimat(ones(m,n).*.asciimat(M))
  • Si A est de type Double et que m est un vecteur, mtlb_repmat(M,m) peut être remplacé par ones(m(1),m(2),...).*.M
  • Si A est de type Booléen et que m est un vecteur, mtlb_repmat(M,m) peut être remplacé par ones(m(1),m(2),...).*..bool2s(M)
  • Si A est de type String et que m est un vecteur, mtlb_repmat(M,m) peut être remplacé par asciimat(ones(m(1),m(2),...).*.asciimat(M))

    • Attention : mtlb_repmat ne doit pas être utilisée pour une programmation manuelle.
