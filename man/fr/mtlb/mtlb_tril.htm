mtlb_tril - Fonction d'émulation de la fonction Matlab tril

Description

Dans plusieurs cas particuliers, la fonction tril se comporte différemment dans Matlab et dans Scilab :

  • Avec des paramètres d'entrée complexes : La fonction Matlab tril peut être utilisée avec des données complexes mais pas la fonction Scilab.
  • Avec des paramètres d'entrée de type chaîne de cacatères : étant donné que Matlab et Scilab ne considèrent pas les chaînes de caractères de la même façon, la fonction tril ne donne pas le même résutat pour ce type d'entrée dans Scilab et dans Matlab.
  • Avec des paramètres d'entrée booléens : La fonction Matlab tril peut être utilisée avec des données booléennes mais pas la fonction Scilab.

    • La fonction mtlb_tril(x,k) est utilisée par mfile2sci pour remplacer tril(x,k) quand il n'était pas possible de savoir ce qu'étaient les paramètres d'entrée pendant la conversion d'un code Matlab vers Scilab. Cette fonction va déterminer la sémantique correcte pendant l'exécution. Pour obtenir un code plus performant on peut remplacer les appels à mtlb_tril :

  • Si X contient des doubles réels mtlb_tril(x,k) peut être remplacé par tril(x,k)
  • Si X contient des booléens mtlb_tril(x,k) peut être remplacé par tril(bool2s(x),k)

    • Attention : mtlb_tril ne doit pas être utilisée pour une programmation manuelle.
